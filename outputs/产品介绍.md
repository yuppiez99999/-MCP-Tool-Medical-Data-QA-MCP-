# 医疗健康数据 AI 模型系统 · 产品介绍

---

## 一、产品概述

HealthcareAI 是一款面向医疗健康领域的 AI 模型训练与数据资产估值平台，基于自主研发的医疗健康 Token 数据标准，提供数据处理、智能分类、资产估值、合规审计等核心能力，助力医疗机构实现数据资产化，帮助 AI 企业快速获取高质量训练数据。

### 核心能力

| 能力模块 | 功能描述 |
|----------|----------|
| **数据处理** | Token 数据加载、清洗、特征工程 |
| **智能分类** | 多任务学习模型，预测等级、质量、科室 |
| **资产估值** | 多因子估值引擎，精准评估数据资产价值 |
| **合规审计** | 全链路审计溯源，确保数据使用合规 |
| **研究分析** | 跨科室聚合分析、质量分布对比研究 |
| **数据登记** | 北数所数据产品登记报告自动生成 |

## 二、核心功能

### 2.1 数据处理模块

**数据加载**
- 支持超大规模 CSV 分块读取（3000条+）
- 自动构建 token_id 索引，支持随机定位
- 分页、筛选、批量读取统一接口

**特征工程**
- 数据清洗：缺失值填充、异常值处理
- 类别 one-hot 编码：8大科室、8种数据类型
- 质量分标准化：完整/准确/时效/合规四维评分
- 衍生特征：均值、方差、最大值、最小值

### 2.2 智能分类模块

**模型架构**
`
输入特征 (24维)
     │
     ▼
  Linear(128) + BatchNorm + ReLU + Dropout
     │
  Linear(64) + BatchNorm + ReLU + Dropout
     │
  Linear(32) + BatchNorm + ReLU + Dropout
     │
  ┌──┴──┬──┐
  ▼     ▼  ▼
Token 质量分 科室
等级   回归  分类
(2类)  (1值) (8类)
`

**多任务学习**
| 任务 | 类型 | 输出 |
|------|------|------|
| Token 等级预测 | 二分类 | A级/ B级 |
| 质量分回归 | 回归 | 0-100 评分 |
| 科室分类 | 多分类 | 8大科室 |

**模型性能**
- Token 等级预测准确率：99%+
- 质量分回归 MAE：< 1.0
- 科室分类准确率：99%+

### 2.3 资产估值模块

**估值公式**
`
Token 价值 = 基础价值 × 质量系数 × 类别权重 × 稀有度系数

其中：
  - 基础价值: 科室基准单价（放射科 ¥15/条）
  - 质量系数: (quality / 100)²（高质量溢价）
  - 类别权重: 不同科室市场稀缺度权重
  - 稀有度系数: 1 / (类别占比^0.5)
`

**科室基准定价**
| 科室 | 基准单价（元） | 权重 |
|------|---------------|------|
| 放射科 | 15.00 | 1.50 |
| 病理科 | 14.00 | 1.40 |
| 神经内科 | 13.50 | 1.35 |
| 心血管科 | 13.00 | 1.30 |
| 检验科 | 10.00 | 1.00 |
| 骨科 | 9.50 | 0.95 |
| 儿科 | 9.00 | 0.90 |
| 急诊科 | 8.50 | 0.85 |

### 2.4 合规审计模块

**审计追踪**
- 每个 Token 的访问记录（时间、调用方、目的）
- token_id → 实体 → 处理历史全链路追踪
- 合规评分自动校验（≥95 视为合格）

**异常检测**
- 高频访问检测（单位时间超过阈值）
- 异常实体聚合检测
- 风险评分实时计算（0-100）

### 2.5 研究分析模块

**分析能力**
1. **跨科室聚合分析** — 同一实体跨科室 Token 追踪
2. **质量分布对比** — 不同科室/模态质量差异分析
3. **时间序列分析** — Token 时效性对估值影响研究
4. **图谱构建** — 基于实体与科室的关联网络分析

## 四、API 接口

### 4.1 接口概览

| Method | Path | 说明 |
|--------|------|------|
| POST | /api/healthcare/predict-level | 预测 Token 等级 |
| POST | /api/healthcare/predict-quality | 预测综合质量分 |
| POST | /api/healthcare/value-token | 单 Token 估值 |
| POST | /api/healthcare/value-batch | 批量 Token 估值 |
| GET | /api/healthcare/asset-summary | 数据资产全景摘要 |
| POST | /api/healthcare/audit-log | 记录审计日志 |
| GET | /api/healthcare/audit-trace/{token_id} | 审计全链路追踪 |
| GET | /api/healthcare/research/{category} | 研究框架分析 |
| POST | /api/healthcare/generate-registration | 生成北数所登记报告 |

## 五、部署方案

### 5.1 本地部署

`ash
# 安装依赖
pip install -r requirements.txt

# 训练模型
python scripts/train_classifier.py --epochs 30 --batch_size 512

# 启动 API 服务
python api/healthcare_ai_extension.py --port 8001
`

### 5.2 超算部署

支持 SLURM 集群并行处理，100M+ 条 Token 处理仅需 0.6-2.4 小时。

### 5.3 云部署

支持 Docker 容器化部署，可在阿里云、腾讯云等平台快速部署。
