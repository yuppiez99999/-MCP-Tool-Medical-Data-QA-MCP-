# 演示视频脚本 — 医疗数据质量评估 MCP Tool

> 时长：2分30秒  
> 语言：中文  
> 建议录制工具：OBS Studio / Screen Studio

---

## 时间轴

### 0:00-0:15 | 开场 — 问题引入

**画面**: 标题卡片「医疗数据质量评估 MCP Tool — 小X宝医疗黑客松 2026」

**旁白**: 
"医疗机构每天产生海量数据，但这些数据有多少能直接用于AI模型训练？数据质量参差不齐、科室分类混乱、合规性存疑，这是医疗AI面临的三大现实痛点。我们构建了一个基于MCP协议的医疗数据质量评估工具，让任何AI Agent都能一键评估数据质量。"

---

### 0:15-0:45 | 功能演示1 — 质量评估

**画面**: 切换到 Gradio 界面「质量评估」Tab，输入示例JSON数据

**操作**: 
1. 在左侧文本框粘贴包含完整性/准确性/时效性/合规性的记录
2. 选择科室「放射科」
3. 点击「评估数据质量」

**展示结果**: 综合得分、A/B/C/D等级、4维度雷达图、改进建议

**旁白**:
"输入一条或多条数据记录，系统会在毫秒级返回4维度综合评分。每条数据获得完整性、准确性、时效性、合规性四项打分，加权汇总为A到D四级评定，直接告诉你能不能用、怎么改进。"

---

### 0:45-1:15 | 功能演示2 — 科室分类

**画面**: 切换到「科室分类」Tab

**操作1**: 在文本输入框输入 "CT scan of brain showing intracranial hemorrhage"，点击分类

**展示结果**: 主科室=放射科、置信度、备选科室列表、Top关键词

**操作2**: 输入 "ECG shows ST elevation in leads II, III, aVF"，点击分类

**展示结果**: 主科室=心血管科

**旁白**:
"科室自动分类支持两个引擎同时工作——基于480篇医学文献训练的ML模型做文本分类，关键词规则做兜底。输入任意医学文本，系统自动判断属于放射科、心血管科还是其他科室，并给出置信度和备选科室。"

---

### 1:15-1:45 | 功能演示3 — 循证医学检索

**画面**: 切换到「循证医学检索」Tab

**操作**: 输入"deep learning diabetic retinopathy screening"，选择「英文论文」，最大返回10条

**展示结果**: 10条检索结果，含标题、作者、期刊、年份、摘要

**旁白**:
"质量评估不是拍脑袋。当你的数据完整性评分偏低时，系统会自动检索循证医学文献告诉你——根据JAMA发表的多中心研究，影像数据的DICOM元数据完整率应达到95%以上。6大数据库源并行检索，结果直接引用到质量报告中。"

---

### 1:45-2:10 | 技术架构展示

**画面**: 架构图 + 数据统计卡片

**展示内容**:
1. MCP协议层 → 9大工具
2. 数据层 → 500万Token、8科室覆盖
3. 模型层 → TF-IDF+LR科室分类、PyTorch Token分类器
4. 循证层 → KnowS API 6数据源并行

**数据展示**: 
- 500万条Token数据
- 8大科室覆盖
- 8个MCP工具
- 科室分类准确率 >70%
- 单条推理 <10ms

**旁白**:
"技术架构分为四层：MCP协议层提供标准化的Agent调用接口，数据层承载500万条脱敏Token记录，模型层部署了两个分类引擎，循证层通过KnowS API接入6个医学文献数据库。全流程可在ModelScope Space一键部署。"
